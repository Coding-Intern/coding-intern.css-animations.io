CSS Animations

Animation Properties:

CSS uses various animation properties for elements like horizontal lines, such as animation-duration, animation-delay, animation-timing-function, animation-name, animation-direction, animation-fill-mode, animation-iteration-count, animation-play-state, and more, to create perfect animations. Take the horizontal line as an example. The animation name is set to, "line," the animation duration is set to 2 seconds, the animation delay was is to 1 second. These three properties are indeed the most important in creating animations.

Property Value(s) Description
animation-name
  • none
  • <custom-ident>
 Used to set a name for the animation in order to style it with the @keyframes rule. It has an initial value of, "none."
animation-duration
  • auto
  • time set in seconds(s) or milliseconds(ms)
 Specifies the time it may take for the animation to complete. The initial value is 0s.
animation-delay
  • time set in seconds(s) or milliseconds(ms)
 Specifies the delay between the animation load and the start of an animation sequence. It's initial value is automatically set to 0s.
animation-timing-function
  • <easing-function>
  • ease
  • linear
  • ease-in
  • ease-out
  • ease-in-out
  • cubic-bezier(p1, p2, p3, p4)
 Defines how the values between the beginning and end of an animation are calculated. It has an initial value of ease.
animation-iteration-count
  • <number>
  • infinite
 Specifies the number of times the animation sequence plays. It has an initial value of 1.
animation-direction
  • normal
  • reverse
  • alternate
  • alternate-reverse
 Specifies if the animation should play forwards, backwards, or alternate forwards and backwards relative to the animation-iteration-count property. The initial value is set to the keyword, normal.
animation-play-state
  • running
  • paused
 Specifies wether an animation is running or not. It's initial value is set to the, "running" keyword.
animation-fill-mode
  • forwards
  • backwards
  • both
 Specifies how an animation applies styles to the object before and after the execution. It's initial value is set to none.
animation-timeline(experimental)
  • none
  • auto
  • scroll()
  • view()
  • <dashed-ident>
 Used to specify the timeline used to control the progression of an animation. This expiremental property has an initial value of, "auto".
animation
  • <single-easing-function>
  • <single-animation-iteration-count>
  • <single-animation-direction>
  • <single-animation-fill-mode
  • <single-animation-play-state>
 A shorthand property for the following animation properties:
  • animation-delay
  • animation-direction
  • animation-duration
  • animation-fill-mode
  • animation-iteration-count
  • animation-name
  • animation-play-state
  • animation-timeline(experimental and may not work)
  • animation-timing-function
. It is displayed in the following format: duration | easing-function | delay | iteration-count | direction | fill mode | play-state | name. It doesn't necessarily have a default (or initial) value.
